# 第 3 章：Smallville：论文中的实验小镇

## 核心问题

Generative Agents：目标不是构造一个会聊天的角色，而是构造一群能在同一个世界中持续生活、记忆、计划、对话和互相影响的虚拟居民。Smallville 是论文用来观察这种行为的实验环境。它解决的核心问题是：

> 为什么 Generative Agents 论文要建造一个类似 The Sims 的小镇，而不是只做命令行模拟或聊天室实验？

答案很直接：因为智能体的可信行为必须落在具体世界中观察。

 - 如果没有地图，角色就没有位置。
 - 如果没有房间和对象，行动就没有落点。
 - 如果没有时间，计划就没有顺序。
 - 如果没有其他居民，社交传播就不会发生。
 - 如果没有可观察的环境，研究者就无法判断智能体是否真的表现出可信行为。

> Smallville 不是论文的视觉包装，而是生成式智能体架构的实验装置。

## 3.1 小镇作为实验环境

论文选择“小镇”作为实验环境，并不是因为小镇本身浪漫，而是因为小镇具备社会生活的基本要素。一座小镇里有居民。居民有身份、职业、性格、关系和生活习惯。一座小镇里有地点。家、咖啡馆、公园、学校、商店、酒吧，不同地点会触发不同类型的行为。一座小镇里有时间。早晨、上午、午饭、下午、晚上、睡前，每个时段都有不同的生活节奏。一座小镇里有偶遇。两个人只有在相近地点才可能看到彼此、交谈、等待或产生后续关系。一座小镇里有公共事件。派对、选举、展览、讲座、商店活动，这些事件可以被讨论、传播、误解、接受或拒绝。这恰好满足论文想研究的问题：个体层面的记忆、计划和对话，如何在共享环境中形成群体层面的社会现象。如果只做命令行模拟，研究者仍然可以记录文本，但很难直观看到角色是否真的去了某个地方、是否在同一时间聚集、是否因为空间接近而发生互动。如果只做聊天室，研究者可以看到对话，但看不到生活。角色没有上下班，没有回家睡觉，没有路上偶遇，也没有“我正在使用这个对象，你要不要等一下”的空间冲突。Smallville 用一个沙盒环境把这些问题放到一起。

```mermaid
flowchart TD
    Town["Smallville 小镇"] --> Agents["25 个智能体"]
    Town --> Map["地图与房间"]
    Town --> Objects["可交互对象"]
    Town --> Time["时间推进"]
    Town --> User["用户干预"]

    Agents --> Behavior["日程/移动/对话/反思"]
    Map --> Behavior
    Objects --> Behavior
    Time --> Behavior
    User --> Behavior
    Behavior --> Evidence["可观察行为证据"]
```

*图 3-1：Smallville 实验环境构成。小镇不是视觉包装，而是让身份、记忆、计划和社交都能落地的实验装置。*

## 3.2 Smallville 的基本构成

论文中的 Smallville 有 25 个智能体。每个智能体都有自然语言人设，包括身份、背景、生活习惯、当前目标和与其他人的关系。这些智能体不是站在聊天室里轮流说话，而是生活在一个二维沙盒环境中。环境中有不同地点和对象，类似 The Sims 风格的生活模拟界面。智能体会在地图上移动，会进入房间，会和对象发生关系，也会在附近看到其他智能体。Smallville 至少包含五类组成部分。第一，角色。角色是系统行为的主体。每个角色都有初始设定。例如，有人是咖啡馆老板，有人是学生，有人是教授，有人是艺术家，有人正在竞选镇长。角色之间还可能有家庭关系、朋友关系、合作关系或冲突关系。第二，地点。地点让行为有了空间约束。咖啡馆、学校、宿舍、公园、商店、酒吧，这些地点不是背景装饰。角色在哪个地点，会影响它可能做什么，也影响它能遇到谁。第三，对象。对象让行动更具体。床、书桌、冰箱、柜台、钢琴、花园、书架，这些对象承接角色的具体行为。角色不是抽象地“工作”，而是可能在柜台后面工作；不是抽象地“休息”，而是可能在床上睡觉。第四，时间。时间让行为有顺序。早上起床、白天工作、晚上休息。时间还决定记忆的近期性，决定计划是否过期，决定邀请是否仍然有效。第五，用户干预。论文中的系统允许用户通过自然语言对环境或角色进行干预。用户可以设置某些事件，也可以和角色互动。这让 Smallville 不只是一个自动播放的仿真系统，而是一个可交互的 AI 社会原型。这五类元素共同构成了一个“可居住”的环境。智能体的语言生成不再漂浮在文本里，而是被地点、时间、对象和社会关系约束。

## 3.3 25 个智能体的实验价值

论文选择 25 个智能体，不只是为了画面热闹。一个智能体可以展示个体行为，例如一天的日程、记忆和反思。两个智能体可以展示对话和关系。少量智能体可以展示局部互动。但要观察社会传播和群体涌现，就需要更多角色。例如，情人节派对不是伊莎贝拉一个人的行为。它至少包含：

- 发起人产生活动意图。
- 发起人邀请别人。
- 被邀请者记住或拒绝。
- 被邀请者可能再告诉别人。
- 其他角色调整自己的计划。
- 到了时间，部分角色聚集到活动地点。

如果只有两个角色，这个过程会很单薄。25 个角色让信息传播有路径，让社交网络有结构，让“某人知道、某人不知道、某人误解、某人转告、某人到场”这样的差异成为可能。镇长竞选也是一样。山姆竞选镇长这件事，在不同角色那里会产生不同意义。支持者可能鼓励他，反对者可能怀疑他，关心公共事务的人可能分析政策，店主可能关心商业环境，艺术家可能把它变成创作或播客话题。这些差异只有在多角色系统中才有观察价值。所以，25 个智能体不是数量噱头，而是为了让社会现象有足够的传播空间。

## 3.4 John Lin 的一天：个体生活连续性

论文中有一个经典观察：智能体 John Lin 的一天。这个例子展示的是个体层面的可信生活。一个角色不是只在别人问他时才出现，而是会沿着自己的生活节奏行动。他会起床，会做早晨例行事务，会去工作，会和家人或熟人互动，会安排自己的日程。这里真正重要的不是某个动作本身，而是动作之间的连续性。如果一个角色早上刚刚准备早餐，下一步又突然说自己刚起床，行为就会断裂。如果一个角色已经在工作，却又完全忘记今天的工作目标，也会显得不可信。如果一个角色在不同场景中持续体现自己的身份、习惯和关系，观察者就更容易相信它是在“生活”。John Lin 的案例说明，Generative Agents 首先要解决的不是宏大社会现象，而是个体行为连续性。没有个体连续性，就没有群体可信性。论文架构从 memory stream、retrieval、reflection、planning 开始，先解决角色如何维持自己的一天。

## 3.5 Isabella 的情人节派对：协同行动如何出现

论文中最广为人知的例子之一，是 Isabella Rodriguez 举办情人节派对。这个案例之所以重要，是因为它展示了“协同行动”不是直接写死的结果，而是通过多个智能体的局部互动产生。一个中央剧本可以轻易写出：

> 晚上 7 点，所有人到咖啡馆参加派对。

但这不是 Generative Agents 的路线。Generative Agents 的路线是：

1. Isabella 有举办派对的意图。
2. 她在生活过程中遇到别人。
3. 她可能自然提到派对。
4. 别人听到后写入记忆。
5. 有些人接受邀请，有些人拒绝，有些人没有机会知道。
6. 部分人可能把消息传给更多人。
7. 到了活动时间，有些人根据记忆和计划到达地点。

这个过程更接近真实社会活动的形成方式。现实中的聚会也不是总由一个全局调度器安排，而是通过邀请、转告、兴趣、时间冲突和关系网络组织起来。在 Generative Agents 中，伊莎贝拉的角色设定保留了这一事件线索。她的 `currently` 中明确写着，她计划在 2 月 14 日下午 5 点到 7 点于霍布斯咖啡馆举办情人节派对，并正在告诉大家参加。这使得本书第四部分可以直接设计复现实验：观察派对消息是否传播，谁被邀请，谁接受，谁拒绝，谁真的到场。这个例子很适合教学，因为它把论文中的多个关键机制串在一起：

- 初始目标。
- 对话传播。
- 记忆写入。
- 日程调整。
- 空间到场。
- 群体聚集。

```mermaid
flowchart LR
    Intention["伊莎贝拉想办派对"] --> Invite1["邀请身边的人"]
    Invite1 --> Memory["被邀请者写入记忆"]
    Memory --> Forward["有人继续转告"]
    Memory --> Plan["有人调整晚间计划"]
    Forward --> Wider["更多角色知道派对"]
    Plan --> Attend["部分角色到场"]
    Wider --> Attend
```

*图 3-2：情人节派对从个人意图到群体行动。协同行动不是直接写死，而是通过邀请、记忆、转告和计划调整逐步形成。*

## 3.6 Sam 竞选镇长：信息扩散与态度分化

另一个重要案例是 Sam 竞选镇长。这个案例展示的是信息扩散。一个公共事件从候选人那里开始，通过对话传播到小镇居民，并在不同角色那里产生不同态度。这比派对更复杂，因为选举不是单纯活动邀请。它涉及公共事务、个人关系、价值判断和利益关联。

 - 一个角色可能支持 Sam，因为他熟悉 Sam，信任他的品格。
 - 一个角色可能保持中立，只是好奇谁会参加选举。
 - 一个角色可能不喜欢 Sam，因此对他的竞选持保留态度。
 - 一个角色可能从自己的职业视角分析政策。例如店主关心商业环境，学生关心社会公平，数学家关心数据，作家关心公共表达。

这就产生了态度分化。Generative Agents 中同样保留了这条线索：

- 山姆打算竞选地方市长，并正在告诉邻居。
- 汤姆对选举感兴趣，但不喜欢山姆。
- 约翰好奇谁会参加下个月的地方市长选举。
- 拉托亚把地方市长选举作为与别人交谈的核心话题。
- 乔治也经常谈论当地市长选举。

这些初始设定使得本项目非常适合复现论文中的信息扩散问题。这里的重点不只是“选举消息有没有传出去”，而是：

- 谁传播了消息？
- 谁只是接收？
- 谁改变了态度？
- 事实是否保持？
- 是否出现误传？
- 是否从“谁参选”扩展到“社区安全、商业环境、公共活动”等政策话题？

人工社会的研究价值也在这里。

## 3.7 关系形成：熟人不是静态标签

论文还关注关系形成。一个角色和另一个角色的关系，不应该只是初始配置中的标签，而应该受到后续互动影响。在真实生活中，关系会因为共同经历而改变。两个人第一次见面，只能根据身份和场景寒暄。第二次见面，如果能提到上次的话题，就说明记忆起作用了。多次交流后，如果形成共同计划、互相帮助或稳定兴趣连接，就说明关系从单次对话变成了连续社会结构。论文中的 Klaus 和 Maria 案例就体现了这一点。Klaus 不只是根据“谁出现得多”来决定互动对象，而会在反思后识别出谁和自己有更深层的共同兴趣。这个案例对本书很重要，因为它说明 reflection 不只是总结日记，而是会改变社交判断。在 Generative Agents 中，克劳斯和玛丽亚都有适合关系形成实验的背景：

- 克劳斯是社会学学生，研究低收入社区中产阶级化。
- 玛丽亚是物理专业学生，也做 Twitch 直播，喜欢建立联系和探索新想法。
- 两人都与奥克山学院、霍布斯咖啡馆等地点有关。

这为后面的关系形成实验提供了自然材料。

## 3.8 Smallville 的三类涌现行为

综合论文案例，可以把 Smallville 中最重要的涌现行为归纳为三类。

### 第一类是信息扩散。

一个角色知道的消息，通过对话传给另一个角色，再继续传播。镇长竞选和派对邀请都是典型例子。

### 第二类是关系形成。

角色之间不是只靠初始关系互动，而是在后续对话、共同事件和反思中形成更稳定的理解。

### 第三类是协同行动。

多个角色在没有中央剧本强制安排的情况下，围绕一个事件形成某种协调。例如多人知道派对并在相近时间来到咖啡馆。这三类行为对应论文架构的不同部分：

| 涌现行为 | 依赖机制 | 典型案例 |
|---|---|---|
| 信息扩散 | 对话、记忆写入、后续检索 | 镇长竞选、派对消息 |
| 关系形成 | 记忆、反思、关系摘要 | Klaus 与 Maria |
| 协同行动 | 计划、日程修改、空间到达 | 情人节派对 |

这里要注意，“涌现”不是神秘词。它并不表示系统有了意识，也不表示行为无法解释。它的意思是：群体层面的结果不是由一条全局规则直接写死，而是由多个个体在局部环境中的选择叠加形成。Smallville 的实验价值就在这里。

## 3.9 Generative Agents 中的小镇

Generative Agents 将论文中的 Smallville 思想落到了一个中文项目中。项目中同样有一个虚拟小镇，资源主要位于：

```text
generative_agents/frontend/static/assets/village/
```

这个目录主要包含这些内容：

- `maze.json`：后端使用的地图和地址数据。
- `tilemap/tilemap.json`：前端 Phaser 使用的 tilemap 数据。
- `tilemap/*.png`：地图图块资源。
- `agents/*/agent.json`：每个角色的人设和空间记忆。
- `agents/*/texture.png`：角色行走贴图。
- `agents/*/portrait.png`：角色头像。

当前项目内置 25 个角色，例如：

- 伊莎贝拉
- 山姆
- 汤姆
- 约翰
- 拉托亚
- 乔治
- 克劳斯
- 玛丽亚
- 阿伊莎
- 埃迪

这些角色不是只有名字和头像。每个角色都有 `currently`、`scratch`、`spatial` 等字段。它们定义了角色当前状态、年龄、性格、背景、生活习惯、日常计划和已知地点。项目中的 `maze.json` 定义了小镇世界：

- `world` 是 `the Ville`。
- `tile_size` 是 32。
- `size` 是 100 x 140。
- `tile_address_keys` 是 `world / sector / arena / game_object`。

小镇不仅有视觉地图，还有后端可理解的空间结构。智能体可以根据地址寻找地点，根据坐标移动，根据感知范围看到附近事件。这一点和论文中的 Smallville 思想一致：智能体必须被放入一个世界，行为才有边界。

## 3.10 项目回放 观察价值

Generative Agents 不只是运行仿真，还能生成回放和 Markdown 报告。运行结束后，`compress.py` 会生成：

```text
results/compressed/<name>/movement.json
results/compressed/<name>/simulation.md
```

`movement.json` 用于浏览器回放。它记录角色初始位置、每一帧移动、当前位置和行动描述。`simulation.md` 更适合阅读和分析。它按时间线保存角色活动和对话内容，让写作和教学不必只看地图动画，而是可以逐条分析：

- 某个角色什么时候做了什么。
- 谁和谁发生了对话。
- 对话中是否提到派对或选举。
- 某个角色是否改变了计划。
- 某个事件是否传播到了更多人。

这使得 Generative Agents 很适合教学。老师可以先展示回放画面，让学生直观看到小镇在动；然后打开 `simulation.md`，带学生分析行为背后的记忆、计划和对话机制；最后再进入源码，看这些现象是如何被实现的。这种“现象 -> 数据 -> 源码”的路径，比直接讲代码更适合课程。

```mermaid
flowchart LR
    Replay["前端回放<br/>看见角色移动与聚集"] --> Files["结果文件<br/>movement.json / simulation.md"]
    Files --> Evidence["证据定位<br/>谁在何时何地做了什么"]
    Evidence --> Code["源码追踪<br/>感知/记忆/计划/对话"]
    Code --> Explain["解释现象<br/>为什么发生或失败"]
```

*图 3-3：从小镇回放到 simulation.md 再到源码分析。先看现象，再回到文件和源码，读者才不会在抽象机制里迷路。*

## 3.11 Smallville 对后续章节的意义

本章讲 Smallville，不是为了单独介绍论文案例，而是为后面四个部分建立共同场景。第一，它为论文架构提供场景。Memory stream、retrieval、reflection、planning 这些概念，如果脱离小镇，会显得抽象；放回 Smallville，它们就变成了居民生活所需的能力。第二，它为源码精读提供入口。Generative Agents 的 `Maze`、`Tile`、`Spatial`、`Agent`、`Schedule`、`Associate`，都可以从“小镇如何运转”这个问题切入。第三，它为实验复现提供材料。情人节派对、镇长竞选、关系形成，都不是额外编造的实验，而是论文和当前项目共同支持的核心现象。第四，它为前沿升级提供基线。未来我们讨论长期记忆、目标驱动规划、组织化多智能体协作、大规模社会仿真时，都可以回到这座小镇，看哪些能力已经有，哪些能力还缺。所以，Smallville 不是第一部分里的一个故事。它是整本书的共同实验场。

## 3.12 本章小结

Smallville 不是论文里的装饰场景，而是一个用来观察智能体行为是否可信的实验装置。读者后面读源码、跑实验、看回放，都要回到这个判断：角色是不是在一个具体世界里持续生活。

| 本章内容 | 核心结论 |
| --- | --- |
| Smallville 的作用 | 小镇让智能体的语言、计划和社交落到地图、时间、地点和对象上。 |
| 五类环境元素 | 角色、地点、对象、时间和用户干预，共同构成可观察的生活环境。 |
| 25 个智能体 | 数量不是噱头，而是给信息扩散、关系形成和协同行动留下空间。 |
| John Lin 的一天 | 个体生活连续性是群体可信性的基础。 |
| 情人节派对 | 协同行动可以从个人意图、邀请、记忆和计划调整中逐步形成。 |
| 镇长竞选 | 同一事件进入不同角色的记忆后，会产生不同态度和传播路径。 |
| Klaus 与 Maria | 关系不是静态标签，反思会改变角色对他人的理解。 |
| Generative Agents 的继承 | `village`、`maze.json`、角色配置、回放和 `simulation.md`，都服务于同一个小镇实验思想。 |

下一章进入 Persona 与人物定义。先看清每个居民是谁、有什么习惯、关系和当前目标，后面再讲 memory stream 时，读者才能理解那些经验记录到底属于谁、为什么重要。
