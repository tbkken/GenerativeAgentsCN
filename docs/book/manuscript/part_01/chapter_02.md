# 第 2 章：论文的核心问题：如何构造一个可信的人工社会

## 本章要解决的问题

上一章我们讲到，Generative Agents 的关键不在于“让模型说得像人”，而在于“让智能体在一个世界中持续表现出可信行为”。

这一章要继续往下追问：

> 如果我们想构造一个可信的人工社会，仅仅让多个大语言模型角色互相聊天，为什么远远不够？

这个问题是整篇论文的真正起点。因为“一个会聊天的 AI 角色”和“一群会生活的虚拟居民”，中间隔着很多系统问题：

- 角色是否记得自己是谁？
- 角色是否记得刚刚发生过什么？
- 角色是否有今天的计划？
- 角色是否会因为别人的话改变行动？
- 信息是否能从一个角色传到另一个角色？
- 多个角色局部互动之后，是否能形成群体层面的现象？

如果这些问题没有解决，我们得到的只是群聊，而不是人工社会。

## 2.1 群聊不是社会

想象一个最简单的系统：我们创建 25 个角色，每个角色都有名字和人设，然后让它们在一个聊天室里轮流说话。这个系统看起来已经有“多智能体”的样子了，因为确实有很多角色，也确实有对话。

但它很快会暴露问题。

第一，角色没有稳定生活。

聊天室里的角色主要在说话。它们没有真正的地点，没有一天的日程，没有“正在做什么”的状态，也没有行动对环境造成的后果。一个角色说“我要去咖啡馆”，但系统并不一定真的把它移动到咖啡馆；它说“我下午 5 点参加派对”，后面也不一定会在 5 点出现在派对地点。

第二，角色没有可靠记忆。

如果系统只把最近几轮聊天塞进上下文，角色可能记得刚刚发生的事，却很难稳定记住几个小时前甚至前一天发生的事。更糟糕的是，模型可能会补全出并不存在的经历。

第三，角色没有独立计划。

如果每一轮都问模型“你现在想说什么”，角色就会被当前对话牵着走。它可能一直聊天，忘记自己本来要工作、学习、吃饭或休息。真实的人不会每一秒都重新发明人生，大多数行为来自习惯、日程和环境约束。

第四，角色之间的影响无法沉淀。

社会不是对话的堆叠。社会关系需要沉淀：谁告诉了谁什么，谁相信了谁，谁对谁产生好感，谁因为某件事改变了后续安排。如果对话结束后什么都不进入记忆和计划，群体行为就不会积累。

因此，群聊系统最多能产生“即时互动”，却很难产生“持续生活”。而 Generative Agents 关心的正是后者。

## 2.2 单轮聊天的根本局限

大语言模型最自然的使用方式是单轮或多轮聊天。用户输入问题，模型给出回答。这个模式很强大，但它默认把世界压缩成文本上下文。

对于问答任务，这没有问题。对于写作、总结、翻译、代码生成，这也很有效。但如果我们想模拟一个小镇的一天，单轮聊天就不够了。

因为生活不是一次回答。

生活有时间。

一个人早上做的事会影响中午的状态，中午遇到的人会影响下午的计划，下午收到的邀请会影响晚上的行动。如果系统无法记录时间，就无法判断某个行为是否合理。早上 7 点准备早餐和凌晨 2 点准备早餐，意义完全不同。

生活有空间。

人在教室、咖啡馆、公园和卧室里会做不同的事。两个人是否能聊天，也取决于他们是否在附近。没有空间约束，角色就可能像漂浮在文本中的声音，随时和任何人说话。

生活有记忆。

一个人会被过去塑造。昨天的对话、刚才的邀请、上周形成的关系，都会影响今天的反应。没有记忆，角色每次出现都像重新加载。

生活有计划。

人通常不是完全随机行动。即使计划会被打断，它仍然提供行为骨架。没有计划，角色可能显得热闹，但缺少生活节奏。

生活有社会传播。

一个人知道的信息会通过对话传给另一个人。信息传播之后，又会改变后者的记忆、态度和行为。单轮聊天很难自然表达这种跨人、跨时间的传播链。

所以，论文要解决的不是“如何让模型回答更好”，而是“如何为模型提供一个能维持生活连续性的外部架构”。

这句话非常关键：

> 大语言模型提供语言推理能力，但人工社会需要系统架构。

## 2.3 可信人工社会需要三种连续性

要让虚拟居民看起来像在生活，至少需要三种连续性：身份连续性、记忆连续性和计划连续性。

### 身份连续性

身份连续性指的是：角色在不同场景中都表现出相对稳定的人设。

例如，伊莎贝拉是咖啡馆老板，友好、外向、好客。这个设定不应该只在自我介绍时出现，而应该持续影响她的行为：

- 她会去霍布斯咖啡馆。
- 她会关注顾客。
- 她会自然地邀请别人参加活动。
- 她会把咖啡馆看成社交场所。

山姆是一名退役军官，正在竞选地方市长。这个设定也应该持续发挥影响：

- 他会在对话中提到社区和竞选。
- 他会关心邻居对公共事务的看法。
- 他可能在公园或咖啡馆传播自己的竞选计划。

如果角色不能维持身份连续性，那么即使语言流畅，也会让人觉得不可信。

在 GenerativeAgentsCN 中，身份连续性主要来自角色的 `agent.json`，尤其是：

- `currently`
- `scratch.innate`
- `scratch.learned`
- `scratch.lifestyle`
- `scratch.daily_plan`

这些字段会通过 `Scratch._base_desc()` 进入多种 prompt，影响角色的日程、行动和对话。

### 记忆连续性

记忆连续性指的是：角色会把过去发生的事情带入未来判断。

如果伊莎贝拉邀请了阿伊莎参加情人节派对，阿伊莎后面应该有可能记住这件事。她可能接受邀请，可能拒绝，可能告诉别人，也可能在晚上到达咖啡馆。如果她完全忘记这件事，行为就会断裂。

如果汤姆不喜欢山姆，而山姆正在竞选地方市长，那么汤姆在讨论选举时不应该像一个没有过去态度的人。他可以保持礼貌，也可以承认某些政策有价值，但他的既有态度应该在语言和行动中留下痕迹。

记忆连续性不是简单保存聊天记录。聊天记录只是记忆的一种。更完整的记忆应该包括：

- 我看见了什么。
- 我做了什么。
- 谁和我说了什么。
- 我从这些事情中总结出了什么。
- 哪些事情对我重要。

论文中的 memory stream 正是为了解决这个问题。GenerativeAgentsCN 中的 `Associate`、`Concept`、`Event` 和 LlamaIndex 索引，就是这一思想的工程实现。

### 计划连续性

计划连续性指的是：角色不是每一步临时乱动，而是围绕日程和目标行动。

计划不等于死板。真实生活中，计划经常被打断。你本来准备学习，朋友突然来找你聊天；你本来要去咖啡馆，路上遇到熟人；你本来要工作，收到活动邀请后改变晚上的安排。

因此，可信智能体需要两种能力同时存在：

1. 有计划。
2. 能根据环境变化调整计划。

只有计划，没有反应，角色会像机械执行脚本。

只有反应，没有计划，角色会像随机聊天对象。

论文中的 planning 和 reacting 正是为了解决这个张力。GenerativeAgentsCN 中的 `Schedule`、`Agent.make_schedule()`、`Agent._reaction()`、`Agent._chat_with()` 和 `Agent._wait_other()`，共同承担这个任务。

## 2.4 多智能体互动为什么会带来级联效应

单个智能体的行为连续性已经不容易，多智能体系统更复杂，因为一个智能体的行为会成为另一个智能体的环境。

这句话需要拆开看。

在普通单人聊天中，模型只需要回应用户。用户说什么，模型答什么。上下文边界相对清楚。

在多智能体小镇中，A 的行为会被 B 观察到，B 会把它写入记忆；B 后来和 C 聊天，可能把这件事告诉 C；C 因此改变计划，到了某个地点；C 的出现又被 D 看到，D 再产生新的反应。

这就是级联效应。

级联效应让人工社会变得有趣，也让它变得难控制。

以情人节派对为例：

1. 伊莎贝拉想在霍布斯咖啡馆办派对。
2. 她遇到亚当，邀请他参加。
3. 亚当因为写书婉拒。
4. 伊莎贝拉遇到阿伊莎，再次邀请。
5. 阿伊莎接受，并计划带文学故事来分享。
6. 阿伊莎后面可能把派对告诉别人。
7. 到了傍晚，部分角色可能出现在咖啡馆。

这里没有一个中央导演逐条安排所有人。事件从一个人的当前状态开始，通过对话、记忆和计划在小镇中扩散。

镇长竞选也是类似。

山姆的竞选意图存在于他的 `currently` 中。其他角色对选举的兴趣存在于各自设定中。汤姆还带有“不喜欢山姆”的初始态度。这样一来，同一个公共事件进入不同角色的记忆后，可能产生不同解释：

- 詹妮弗可能支持山姆。
- 约翰可能好奇候选人是谁。
- 拉托亚可能把选举当成谈话核心。
- 乔治可能从数据和数学角度分析选情。
- 汤姆可能从商店经营和个人好恶出发评价山姆。

这就比“大家都听到一个消息”复杂得多。真正值得观察的是：同一事件如何在不同身份和记忆中被加工。

## 2.5 可信人工社会不是全局剧本

这里要避免一个误解：人工社会中的复杂行为，不应该靠一个隐藏的全局剧本来伪装。

如果我们直接写死：

- 17:00 所有人去咖啡馆参加派对。
- 山姆告诉所有人自己竞选。
- 汤姆说不喜欢山姆。
- 克劳斯和玛丽亚建立关系。

这样当然也能产生想要的画面。但它不是论文追求的方向。

论文更关心的是：能否通过个体机制，让群体现象自然出现。

这并不是说系统完全没有设计。恰恰相反，系统设计非常重要。角色人设、地图、记忆、检索、反思、计划、对话 prompt，这些都是设计。区别在于：

- 全局剧本直接规定结果。
- 智能体架构规定个体如何感知、记忆、思考和行动。

前者是导演式控制。

后者是机制式生成。

Generative Agents 的魅力就在于机制式生成。设计者不直接写“谁一定参加派对”，而是设计出一个系统，使得“某些人因为知道派对、觉得感兴趣、有时间、地点可达，于是选择参加”成为可能。

这种机制式生成也意味着失败是正常的。

有人可能没听说派对。有人可能听说了但忘了。有人可能口头答应却没到场。有人可能误解时间或地点。传统 demo 往往隐藏这些失败，但一本严肃的书必须把这些失败讲出来，因为它们恰恰暴露了系统边界。

本书第四部分设计实验时，会特别关注这些失败模式。

## 2.6 大语言模型在人工社会中扮演什么角色

在 Generative Agents 中，大语言模型不是整个世界。

它更像每个居民脑中的自然语言推理器。

系统把角色背景、当前状态、相关记忆、地点、时间和对话历史组织成 prompt，然后让模型完成一些局部任务：

- 这个角色几点起床？
- 这个角色今天的大致日程是什么？
- 当前计划应该拆成哪些小行动？
- 这个行动应该发生在哪个地点？
- 眼前这个人是否值得聊天？
- 如果聊天，应该说什么？
- 这次对话是否应该结束？
- 这个事件有多重要？
- 最近经历能总结出什么 insight？

这些任务都适合 LLM，因为它们依赖自然语言、常识和上下文。

但 LLM 不负责所有事情。

地图不是 LLM 想象出来的，而是 `maze.json` 和 `tilemap.json` 定义的。

时间不是 LLM 随口说的，而是 `Timer` 推进的。

路径不是 LLM 规划坐标，而是 `Maze.find_path()` 计算的。

记忆不是全部塞进 prompt，而是 `Associate` 检索出来的。

回放不是 LLM 生成视频，而是 checkpoint 经过 `compress.py` 变成 `movement.json`，再由 Phaser 展示。

这就是架构的意义。

如果说 LLM 提供的是“语言和常识”，那么系统架构提供的是“边界和连续性”。两者缺一不可。

## 2.7 可信与可控之间的张力

构造人工社会还有一个重要张力：可信和可控不完全一致。

越是手写脚本，系统越可控。设计师可以保证每个角色在指定时间做指定动作。但这种系统容易显得死板。

越是依赖生成式智能体，行为越灵活。角色可能产生设计师没有写过的对话、计划和互动。但这种系统也更不可控。

例如，伊莎贝拉邀请别人参加派对时，模型可能生成很自然的邀请，也可能忘记说清时间；可能有人合理拒绝，也可能所有人都过度热情地接受；可能话题顺利传播，也可能对话偏到完全无关的方向。

这不是简单 bug，而是生成式系统的基本特征。

因此，本书后面会坚持一个原则：

> 不只展示系统能做什么，也要记录它什么时候失败、为什么失败、如何评价失败。

这也是为什么论文中的 evaluation 很重要。论文没有只展示 Smallville 的故事，而是通过 controlled evaluation、ablation study 和 end-to-end evaluation 来检验不同架构组件的贡献。

GenerativeAgentsCN 作为中文工程项目，也需要类似的评价意识。我们不能只说“角色聊起来了”，还要问：

- 它是否记得过去？
- 它是否符合人设？
- 它是否按计划行动？
- 它是否把承诺转化成行动？
- 它是否在对话中传播了事实？
- 它是否产生了幻觉或错误记忆？

这些问题会在第 26 章系统展开。

## 2.8 从人工社会问题回到 GenerativeAgentsCN

现在我们可以回头看 GenerativeAgentsCN 的工程结构。

这个项目之所以值得写成书，不是因为它代码量巨大，而是因为它覆盖了人工社会所需的关键部件：

- 角色设定：`agent.json`
- 地图世界：`maze.json`、`Maze`、`Tile`
- 时间系统：`Timer`
- 事件表示：`Event`
- 当前行动：`Action`
- 日程系统：`Schedule`
- 空间记忆：`Spatial`
- 长期记忆：`Associate`
- 向量检索：`LlamaIndex`
- 提示词工厂：`Scratch`
- 模型适配：`LLMModel`
- 仿真入口：`start.py`
- 数据压缩：`compress.py`
- 可视回放：`replay.py`、`live.py`

这些模块正好对应本章讨论的人工社会需求：

| 人工社会需求 | GenerativeAgentsCN 模块 |
|---|---|
| 身份连续性 | `agent.json`、`Scratch._base_desc()` |
| 记忆连续性 | `Associate`、`Concept`、`Event` |
| 计划连续性 | `Schedule`、`Action` |
| 空间约束 | `Maze`、`Tile`、`Spatial` |
| 时间推进 | `Timer`、`stride` |
| 社会互动 | `_reaction()`、`_chat_with()`、`conversation.json` |
| 行为观察 | checkpoint、`simulation.md`、`movement.json` |

后面读源码时，我们不应该把这些模块当成孤立工具。它们共同服务一个问题：让多个由 LLM 驱动的角色在同一个世界中持续生活。

## 2.9 本章小结

本章讲清了论文的核心问题：

1. 多个 LLM 角色互相聊天，不等于一个人工社会。
2. 可信人工社会需要身份连续性、记忆连续性和计划连续性。
3. 多智能体互动会带来级联效应，一个人的话语可能影响另一个人的记忆和第三个人的行动。
4. 论文追求的是机制式生成，而不是全局剧本。
5. LLM 在系统中提供自然语言推理能力，但世界、时间、路径、记忆和回放都需要外部架构。
6. 可信和可控之间存在张力，因此评价和失败分析必须进入本书。
7. GenerativeAgentsCN 的模块结构正好对应人工社会的关键需求。

下一章我们进入论文中的实验环境 Smallville。那里不是一个简单的游戏画面，而是论文用来观察智能体生活、社交传播和群体涌现的实验装置。
