## 二、全书定位

### 暂定书名

主标题：

```text
从聊天机器人到虚拟小镇
```

副标题：

```text
Generative Agents 论文精读、GenerativeAgentsCN 源码解析与智能体系统实战
```

备选书名：

1. `会生活的 AI：Generative Agents 论文、源码与智能体小镇实战`
2. `构建生成式智能体小镇：从论文到源码到前沿升级`
3. `GenerativeAgentsCN 实战：从 Smallville 到中文 AI 小镇`

### 一句话定位

这是一本面向教学和项目实战的生成式智能体系统书，先讲透 Generative Agents 原论文，再读懂 GenerativeAgentsCN 源码，最后带读者复现实验、改造项目，并理解 2023-2026 年智能体领域的前沿演进。

### 核心卖点

1. 不只讲概念。
   - 讲论文，也讲源码。
2. 不只跑 demo。
   - 讲实验、评价和失败模式。
3. 不只复述 2023 年论文。
   - 补上 2023-2026 年前沿发展。
4. 不只会用项目。
   - 带读者修改角色、事件、记忆、规划、评价和升级路线。
5. 适合教学。
   - 每部分都能拆成课程单元、实验任务和项目作业。

## 三、目标读者画像

### 第一读者：课程设计者和教师

他们需要：

- 能讲清 Generative Agents 论文。
- 能带学生运行中文项目。
- 能设计实验作业。
- 能把源码拆成教学模块。
- 能讨论智能体系统的边界和伦理。

写作策略：

- 每章要有清晰教学目标。
- 每个实验要能变成课堂或课程项目。

### 第二读者：AI / 大模型方向学生

他们需要：

- 理解 Agent 架构。
- 理解记忆、反思、规划、多智能体互动。

## 四、写作风格

### 应该像什么

项目实战书，但有论文深度。

具体风格：

- 先提出问题。
- 再解释论文思想。
- 再落到当前项目。
- 再读关键源码。
- 再做实验观察。
- 最后总结可改造方向。

### 不应该像什么

1. 不像论文翻译。
   - 不逐段翻译论文。
2. 不像 API 文档。
   - 不只列函数参数。
3. 不像 demo 教程。
   - 不止于“运行成功”。
4. 不像前沿综述。
   - 不堆论文名。
5. 不像营销文章。
   - 不神化智能体。

## 十二、不做事项

### 不做完整论文翻译

原因：

- 这本书不是论文译本。
- 目标是理解、实现、实验和扩展。

### 不做完整前端开发书

原因：

- Phaser 前端只服务回放。
- 不应让前端细节喧宾夺主。

### 不做大规模 Agent 平台重构

原因：

- 本书核心是基于当前项目渐进改造。
- 大重构会偏离 GenerativeAgentsCN。

### 不承诺仿真能预测现实社会

原因：

- 生成式智能体是实验装置。
- 不能把小镇结果当作真实社会预测。

### 不只展示成功案例

原因：

- 失败模式是评价和改造的关键。
- 用户已明确要求不要偷懒，必须把问题讲透。

## 十三、质量标准

每一章写完后都要检查：

1. 是否回答了本章核心问题。
2. 是否连接了论文概念。
3. 是否落到了项目源码或实验数据。
4. 是否有图表或表格帮助理解。
5. 是否说明了失败模式。
6. 是否有明确下一章过渡。
7. 是否避免了神化智能体。
8. 是否能作为教学材料使用。

1. 是否给出文件路径。

4. 是否讲清状态变化。
5. 是否说明 prompt 的作用。
6. 是否指出与论文设计的差异。

每个实验章节还要检查：

1. 是否可运行。
2. 是否有命令。
3. 是否有证据材料。
4. 是否有评价指标。
5. 是否有失败模式。
6. 是否能生成教学作业。
